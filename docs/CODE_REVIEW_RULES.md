# 代码审查规则

## 审查优先级
1. 业务正确性
2. 数据安全
3. 安全性
4. 权限和租户
5. 事务安全
6. API 和前端兼容
7. 测试和验证
8. 性能
9. 可维护性
10. 代码风格

## 阻塞问题
出现以下情况必须拒绝变更：
- 代码无法编译。
- 业务规则是猜测出来的。
- 状态流转错误。
- 权限或租户隔离缺失。
- 多表写入缺少事务。
- SQL 可能破坏数据。
- 表结构变化缺少 migration SQL。
- API 响应结构破坏兼容性。
- 前端缺失关键状态处理。
- 暴露敏感数据。
- 引入未经批准的依赖。
- 修改无关文件。

## 输出格式
1. 阻塞问题
2. 重要问题
3. 次要建议
4. 表结构/字段变化审查
5. API/前端契约审查
6. 安全和性能审查
7. 验证步骤
8. 风险总结
